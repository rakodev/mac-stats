# MacStats Follow-Ups

Follow-ups are ideas worth revisiting later, but not committed backlog work yet. They may need user demand, clearer evidence, a narrower scope, or validation that they still fit MacStats' minimal CPU, Memory, and Disk menu bar focus.

Promote an item to `BACKLOG.md` only when it has a clear problem statement, priority, evidence, and acceptance criteria.

## Future Improvements

- [ ] Minimal notification when memory or disk crosses a threshold.
  - Why later: The existing project guide identifies threshold notifications as a possible extension point, but useful defaults, notification preferences, and low-overhead behavior need product validation before this becomes committed work.
- [ ] Optional toggle to hide disk in width-sensitive menu bars.
  - Why later: The existing project guide says to add this only if strongly requested; there is not yet clear evidence that disk display width is a recurring user problem.
